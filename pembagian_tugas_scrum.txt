============================================================
  SCRUM ARTIFACTS & EVENTS
============================================================

ARTIFACTS:
  1. Product Backlog  : Daftar fitur prioritas (dikelola Rayhan/PO).
  2. Sprint Backlog   : Tugas yang dipilih untuk dikerjakan Sprint ini.
  3. Increment        : Aplikasi PHP yang berfungsi di akhir setiap Sprint.

EVENTS:
  1. Sprint Planning  : Menentukan fitur PHP yang akan dikerjakan.
  2. Daily Stand-up   : Update harian: kemarin, hari ini, hambatan.
  3. Sprint Review    : Demo fitur PHP yang telah selesai ke PO.
  4. Retrospective    : Evaluasi proses kerja tim untuk Sprint berikutnya.

DEFINITION OF DONE (DoD):
  - Kode PHP sudah berjalan tanpa error di XAMPP (localhost).
  - Fitur sudah diuji oleh Hafiyan (QA) dan lulus uji fungsional.
  - Kode sudah di-commit dengan pesan Conventional Commits ke GitHub.
  - Tidak ada data sensitif (password, token) dalam kode yang di-push.

============================================================
  KONVENSI COMMIT (Conventional Commits)
============================================================
  feat(scope)  : Fitur baru PHP yang ditambahkan
  fix(scope)   : Perbaikan bug pada PHP/SQL
  docs(scope)  : Perubahan dokumentasi atau .txt
  style(scope) : Perubahan CSS / tampilan tanpa logika
  refactor     : Refaktor kode PHP tanpa mengubah fungsi
  chore        : Pemeliharaan (konfigurasi, .htaccess, dsb.)

  Contoh:
  feat(member): add project listing page with PHP session check
  fix(auth): resolve redirect loop on login.php
  feat(db): add foreign key constraint to task_category_map
  docs(admin): update sprint backlog and task assignment
